Error Codes Handling

Note: The error codes are returned back to the application, through the delegate function 

func miaSDKPaymentFailed(_ MiaSDKController: MiaSDKController, withError error: MIAError) {}

In the error case, the returned error - MIAError - object will be non nil, which has getters for error code and error message. The error codes are mapped inside an enum MIAErrorCode, but each explanation message can be retrieved by calling localizedDescription. If you want custom error messages, you can create your custom map based on the codes.

Note:

We recommend that you keep all error codes internal without showing them to your users. Instead, you should provide generic messages encouraging them to retry the transaction, to select another payment method, or to contact your customer service. There are handled internal SDK errors, such as:

  • GENERIC_ERROR, code 300 : This occurs inside the WebView communication towards Easy Platform. It means the payment authorization process has failed.

    Note: In this case, the next step you need to do is call Reporting API, in order to find the cause of the failed payment. According to it, you will show the proper message for the user.
